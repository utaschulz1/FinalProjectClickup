Clickup API

This Clickup API documentation created in Swagger is the final project for the Software Documentation course with Jordan Stanchev for the TCLoc master's program. Clickup is a projectmanagement tool. This API documentation introduces a get method to query all fields with their values from on specific task, a get method to query all custom fields that were created in a specific list without their values, and a post method to change/set field values in a task by using field IDs.

Admins

Up 
post /task/{task_id}/field/{field_id}
sets a custom field value (customFieldvalue)
This methods changes or sets a value of a specific field of a task.

Path parameters

task_id (required)
Path Parameter — Enter a task ID to identify the task where the custom field lives in
field_id (required)
Path Parameter — Enter a field ID. You can retrieve the field ID on the task object from the get task route.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body SetCustomfieldValue (optional)
Body Parameter — The value to be set in the custom field

Query parameters

custom_task_ids (optional)
Query Parameter — If you want to reference a task by it's custom task id, this value must be true
team_id (optional)
Query Parameter — Only used when the parameter is set to custom_task_ids=true

Responses

201

task created

400

invalid input, object invalid

409

an existing task already exists

Up 
get /list/{list_id}/field
gets accessible custom fields in one list (getCustomfields)
This method get all custom fields that were created in one specific list. This can be useful for retrieving IDs to set field values.

Path parameters

list_id (required)
Path Parameter — pass a list id to call one task

Return type

array[GetCustomfield]

Example data

Content-Type: application/json
[ {
  "id" : 0
}, {
  "id" : 0
} ]

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

queried task

400

bad input parameter
Up 
get /task/{task_id}
calls complete task with all fields (getTask)
This method gets all fields and their values of one specific task, including existing custom fields and attachements.

Path parameters

task_id (required)
Path Parameter — pass a task id to call one task

Query parameters

custom_task_ids (optional)
Query Parameter — If you want to reference a task by its custom task id, this value must be true
team_id (optional)
Query Parameter — Only used when the parameter is set to custom_task_ids=true
include_subtasks (optional)
Query Parameter — Include subtasks, default false

Return type

array[getTask]

Example data

Content-Type: application/json
[ {
  "id" : 0
}, {
  "id" : 0
} ]

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

queried task

400

bad input parameter
